# 前端系统字典与枚举接入

## 1. 先区分三类选项

| 类型 | 数据来源 | 前端处理 |
|---|---|---|
| 动态字典 | `/api/v1/system/dicts/**` | 运行时请求，允许后台维护，不在前端复制一份长期常量 |
| 固定枚举 | OpenAPI 字段和本文映射 | 放入前端类型与显示映射；遇到未知值显示原值，不静默丢弃 |
| 关联选项 | 部门、角色、菜单、任务 Handler 等资源接口 | 从对应资源接口实时读取，不能当成字典硬编码 |

动态字典用于稳定的“值、中文名称、颜色、排序”配置。用户、部门、角色、菜单等会持续增删的实体不是字典。

## 2. 动态字典读取接口

### 2.1 普通业务页面只需要两个 GET

| 方法与路径 | 用途 | 权限 |
|---|---|---|
| `GET /api/v1/system/dicts/{dictCode}/items` | 读取单个启用字典的启用项 | 登录即可 |
| `GET /api/v1/system/dicts/items?codes=code1,code2` | 批量读取多个字典，减少首屏请求 | 登录即可 |

单个字典响应中的 `data`：

```json
[
  {
    "dictCode": "async_task_status",
    "label": "运行中",
    "value": "RUNNING",
    "tagType": "warning",
    "sortNo": 20
  }
]
```

批量接口响应中的 `data`：

```json
{
  "async_task_status": [
    {
      "dictCode": "async_task_status",
      "label": "成功",
      "value": "SUCCESS",
      "tagType": "success",
      "sortNo": 30
    }
  ]
}
```

建议类型：

```ts
export type DictItem = {
  dictCode: string;
  label: string;
  value: string;
  tagType?: "primary" | "success" | "warning" | "danger" | "info" | string;
  sortNo?: number;
};
```

只返回启用的字典类型和启用的字典项，并按照 `sortNo`、内部 ID 升序排列。编码不存在、类型停用或没有启用项时返回空数组，不应由前端伪造默认项。

### 2.2 实现字典管理页面时使用完整接口

当前已包含以下 20 个字典操作：

| 方法 | 路径 | 用途 |
|---|---|---|
| GET | `/api/v1/system/dicts/{dictCode}/items` | 单字典消费 |
| GET | `/api/v1/system/dicts/items` | 多字典批量消费 |
| GET / POST | `/api/v1/system/dicts/types` | 分页查询、新增字典类型 |
| GET / PUT / DELETE | `/api/v1/system/dicts/types/{dictTypeId}` | 详情、修改、删除字典类型 |
| GET | `/api/v1/system/dicts/types/{dictTypeId}/disable-impact` | 停用前影响检查 |
| GET | `/api/v1/system/dicts/items/manage` | 查询包含停用项在内的管理列表 |
| POST | `/api/v1/system/dicts/items` | 新增字典项 |
| PUT / DELETE | `/api/v1/system/dicts/items/{dictItemId}` | 修改、删除字典项 |
| POST | `/api/v1/system/dicts/items/sort` | 保存字典项排序 |
| POST | `/api/v1/system/dicts/items/batch/status` | 批量启停 |
| POST | `/api/v1/system/dicts/items/batch/delete` | 批量删除 |
| POST | `/api/v1/system/dicts/refresh` | 清空后端字典缓存 |
| GET | `/api/v1/system/dicts/template` | 下载 CSV 导入模板 |
| POST | `/api/v1/system/dicts/import` | 导入 CSV |
| GET | `/api/v1/system/dicts/export` | 导出 CSV |
| GET | `/api/v1/system/dicts/logs` | 查询变更记录 |

除两个消费 GET 外，管理操作要求 `system:dict:view`。当前基座按页面级权限控制，尚未拆分新增、修改、删除等按钮级权限。

## 3. 当前动态字典快照

### 3.1 独立运行前端需要关注

| 字典编码 | 名称 | 值与中文 | 颜色 |
|---|---|---|---|
| `async_task_status` | 异步任务状态 | `PENDING` 等待；`RUNNING` 运行中；`SUCCESS` 成功；`FAILURE` 失败；`CANCELED` 已取消 | info；warning；success；danger；info |

当前业务示例另有：

| 字典编码 | 名称 | 值与中文 |
|---|---|---|
| `station_status` | 工站状态 | `ENABLED` 启用；`DISABLED` 停用 |

`station_status` 属于业务字典，不是系统模块的固定依赖。以后真实业务模块应在冻结契约中逐项声明所需字典编码。

### 3.2 开发中心专用字典

方向 2 的独立前端不实现 `/api/v1/dev/**`，因此以下字典无需接入：

| 字典编码 | 当前值 |

| `generation_review_status` | `PENDING`、`APPROVED`、`REJECTED` |
| `generation_build_status` | `NOT_STARTED`、`AWAITING_CONFIRMATION`、`PASSED`、`FAILED` |
| `generation_acceptance_status` | `PENDING`、`PASSED`、`FAILED` |
| `generation_run_status` | `RUNNING`、`SUCCEEDED`、`CONFLICT`、`FAILED` |
| `generation_artifact_status` | `PLANNED`、`WRITTEN`、`UNCHANGED` |
| `generation_risk_level` | `NONE`、`LOW`、`MEDIUM`、`HIGH` |

## 4. 系统模块固定枚举

以下字段当前不是动态字典，需要前端按契约维护显示映射：

| 模块 / 字段 | 值 | 中文显示 | 备注 |
|---|---|---|---|
| 用户、角色、部门、菜单、字典、编码规则 `status` | `1` / `0` | 启用 / 禁用 | 编码规则页面当前使用“停用”，语义等同禁用 |
| 任务配置 `status` | `1` / `0` | 运行 / 暂停 | 不要显示成用户启用状态 |
| 菜单 `menuType` | `DIR` / `MENU` | 目录 / 菜单 | 当前没有按钮类型 |
| 菜单 `visible` | `true` / `false` | 可见 / 隐藏 | 与 `status` 含义不同 |
| 参数、字典 `configScope` / `dictScope` | `FOUNDATION` / `BUSINESS` | 系统 / 业务 | 系统项受保护，业务项用于项目配置 |
| 参数 `configType` | `string` / `boolean` / `number` / `json` | 原值显示 | 保存前按类型校验内容 |
| 编码规则 `resetScope` | `NONE` / `DAILY` / `MONTHLY` / `YEARLY` | 不重置 / 每天 / 每月 / 每年 | 固定枚举 |
| 字典项 `tagType` | 空、`primary`、`success`、`warning`、`danger`、`info` | 默认、蓝、绿、黄、红、灰 | 表示显示色，不是业务状态 |
| 任务 `jobScope` | `SYSTEM` / `BUSINESS` | 系统任务 / 业务任务 | 系统任务由基座维护 |
| 任务 `sourceType` | `FOUNDATION` / `GENERATOR` / `MANUAL` | 基座内置 / 向导生成 / 手工创建 | 展示来源，通常不可编辑 |
| 任务 `concurrencyPolicy` | `FORBID` / `ALLOW` | 禁止并发 / 允许并发 | 默认 `FORBID` |
| 任务 `failureStrategy` | `FAIL_AND_RECORD` / `RETRY_THEN_FAIL` | 失败并记录 / 重试后失败 | 默认 `FAIL_AND_RECORD` |
| 任务日志 `status` | `RUNNING` / `SUCCESS` / `FAILURE` | 运行中 / 成功 / 失败 | 固定执行状态 |
| 生命周期日志 `status` | `RUNNING` / `SUCCESS` / `FAILURE` | 运行中 / 成功 / 失败 | 与任务日志一致 |
| 登录日志 `successFlag` | `true` / `false` | 成功 / 失败 | 布尔值，不发送字符串 |
| 操作日志 `operationType` | `CREATE` / `UPDATE` / `DELETE` / `OTHER` | 创建 / 更新 / 删除 / 其他 | 由 HTTP 方法归类 |

未知枚举值的统一策略：显示后端原值并使用中性色，同时记录可观测告警；不要显示为空白，也不要阻止整个页面渲染。

## 5. 不是字典的关联选项

| 页面字段 | 正确来源 |
|---|---|
| 用户所属部门 | `GET /api/v1/system/depts` 或 `/depts/tree` |
| 用户角色 | `GET /api/v1/system/roles`；编辑用户时再读取 `/users/{userId}/roles` |
| 角色菜单授权 | `GET /api/v1/system/menus` 与 `/roles/{roleId}/menu-ids` |
| 菜单父节点 | `GET /api/v1/system/menus` 后在前端组树 |
| 任务 Handler 类型 | `GET /api/v1/system/jobs/handlers` |

新增或编辑用户前应刷新部门与角色候选项，不能只使用应用启动时缓存的数据。

## 6. 文件分组固定约定

文件分组不是动态字典，当前约定如下：

| 编码 | 中文 | 是否系统托管 |
|---|---|---:|
| `default` | 手工上传 | 否 |
| `documents` | 文档资料 | 否 |
| `images` | 图片 | 否 |
| `exports` | 导出结果 | 是 |
| `imports` | 导入结果 | 是 |
| `attachments` | 业务附件 | 是 |
| `generated` | 系统生成 | 是 |

普通文件管理页面只允许向非托管分组手工上传。托管分组由导入、导出或业务流程写入；未过期的托管文件不能从普通文件页面误删。

## 7. 缓存与刷新约定

- 后端字典缓存参数为 `foundation.dict.cache-ttl-seconds`，默认 300 秒。
- 字典类型或字典项新增、修改、删除、排序和导入后，后端会清除对应编码缓存。
- `POST /api/v1/system/dicts/refresh` 清空全部后端字典缓存，但不会自动清除独立前端自己的内存或持久化缓存。
- 前端推荐使用“同请求去重 + 最长 5 分钟内存缓存”，不要永久写入 `localStorage`。
- 字典管理成功后，应主动清除客户端对应编码；全量刷新成功后清除全部客户端字典缓存。
- 页面刚进入且依赖多个字典时优先调用批量接口。
- 字典读取失败时保留字段原值并提供重试，不要把接口失败解释为空字典。

当前基座前端提供 `DictSelect`、`DictTag` 和 `useDict` 作为参考。独立前端可以按自己的组件体系实现，但必须保留上述数据语义和失效规则。
