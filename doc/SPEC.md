# 工业系统管理平台前端 SPEC

> 状态：已确认，作为首版开发与逐页验收基线  
> 更新日期：2026-07-15  
> 接口契约：`doc/foundation-frontend-runtime-V3.openapi.json`  
> 字典契约：`前端系统字典与枚举接入.md`

## 1. 项目目标

构建一套纯前端的 RBAC 管理系统，使用后端提供的 JWT、菜单、角色、用户及系统管理接口。权限控制到菜单页面级别，角色通过菜单树配置可访问页面，用户登录后依据其角色动态生成菜单与路由。

系统面向 PC 端运维和管理人员，强调高信息密度、清晰状态、稳定操作反馈及可追踪错误，不做移动端适配。

## 4. 视觉与布局

### 4.1 视觉方向

- 风格：现代工业管理台，冷静、精确、克制。
- 产品名称：工业系统管理平台。
- 主色：`#0d9496`。
- 默认浅色主题，同时支持深色和跟随系统。
- 冷灰背景、白色或深灰内容面板、清晰细边框、轻量阴影。
- 信息密度偏紧凑，避免大面积渐变、装饰性卡片和过度动画。
- 中文界面字体优先使用系统可用的现代中文无衬线字体栈，不从公网加载字体。
- 设计基准宽度为 1440px；仅保证 PC 端完整使用。窄窗口允许表格横向滚动，不实现移动布局。

### 4.2 主框架

- 左侧为可折叠菜单栏。
- 顶部工具栏包含面包屑、主题入口、全屏入口和用户菜单。
- 内容顶部提供多标签页导航。
- 工作台标签固定且不可关闭。
- 动态菜单、页面标签和面包屑使用同一份菜单树生成。

### 4.3 主题定制

主题设置只提供：

- 外观：浅色、深色、跟随系统。
- 主色：默认 `#0d9496`，允许颜色选择器调整。
- 圆角：0px、6px、10px。
- 密度：紧凑、标准。

主题设置写入 `localStorage`，不依赖后端。

## 5. 认证与会话

### 5.1 登录流程

1. 用户提交用户名和密码至 `POST /api/v1/auth/login`。
2. 保存返回的 token、`expiresAt` 和 `refreshExpiresAt` 至 `localStorage`。
3. 使用登录响应中的用户、角色、菜单和权限初始化会话。
4. 生成动态路由后进入工作台；无工作台权限时进入第一个可访问菜单。

### 5.2 会话恢复与刷新

- 应用启动时读取本地 token，并调用 `GET /api/v1/auth/me` 校验和恢复用户会话。
- Axios 自动附加 `Authorization: Bearer <token>`。
- token 临近过期时调用 `POST /api/v1/auth/refresh`。
- 并发请求只允许触发一次刷新，其他请求等待刷新结果。
- 刷新失败或接口返回不可恢复的 401 时清除会话、动态路由、标签页及缓存并进入登录页。
- 退出登录先调用 `POST /api/v1/auth/logout`；无论接口是否成功，本地会话最终都必须清理。

## 6. RBAC、菜单与动态路由

### 6.1 权限边界

- 首版只控制页面是否可访问，不控制页面内按钮。
- 后端仍负责所有接口的最终授权，前端隐藏菜单不视为安全边界。
- 手动访问无权路由时显示 403 页面。

### 6.2 动态路由

- 登录成功及会话恢复后，统一调用 `GET /api/v1/system/menus` 获取动态菜单；登录响应和 `/api/v1/auth/me` 只负责认证及用户会话信息。
- 仅 `status=1` 且 `visible=true` 的可访问菜单进入侧栏；隐藏菜单可作为路由存在，但不显示在侧栏。
- 菜单路由由各级菜单 `path` 逐级拼接，例如父级 `system` 与当前 `user` 生成 `/system/user`。
- 菜单 `component` 只存储 `/system/user` 形式的组件目录。
- 前端通过 `import.meta.glob('/src/views/**/*.vue')` 自动收集页面组件，将后端目录转换为 `/src/views${component}/index.vue` 后加载，不维护手工组件映射表。
- 新增根菜单或根目录时不选择上级目录，请求参数 `parentId` 固定提交为 `0`；选择上级目录时提交对应菜单 ID。
- 找不到组件文件时进入“页面未配置”错误页，并输出包含菜单 ID、路径和组件路径的开发告警。
- 未知、重复或非法路由路径不得导致整棵菜单加载失败。

### 6.3 菜单图标

- `icon` 存储 Ant Design Vue 图标组件名，如 `UserOutlined`。
- 菜单管理提供常用图标搜索选择。
- 空值或未知图标回退为 `AppstoreOutlined`。
- 不渲染后端传入的 SVG 或 HTML 字符串。

## 7. 多标签页

- 首次访问页面时新增标签。
- 支持关闭当前、关闭其他、关闭右侧和刷新当前。
- 页面使用 `KeepAlive` 保留查询条件、分页和滚动位置。
- 关闭标签时同步销毁对应页面缓存。
- 标签列表保存至 `sessionStorage`，页面刷新可恢复，关闭浏览器后不保留。
- 权限或菜单变化后移除已无权访问或已不存在的标签。

### 7.1 内容区滚动规则

- 菜单页面必须撑满顶部工具栏和标签栏以下的可用内容区域，不出现页面级纵向滚动条。
- 页面内容区外边距统一使用紧凑规格：顶部 `8px`、左右 `10px`、底部 `10px`。
- 查询区与页面工具栏固定；表格数据超出可用高度时，仅表格主体内部纵向滚动。
- 表格列超出可用宽度或拖拽增大列宽时，仅表格内部横向滚动，操作列保持固定在右侧。

## 8. 通用页面交互

### 8.1 列表与分页

- 所有扁平列表均使用后端分页。
- 默认每页 20 条，可切换 10、20、50、100 条。
- 显示总记录数。
- 查询条件变化后点击查询时回到第 1 页。
- 翻页和修改每页条数均重新请求后端。
- 不下载全量数据进行前端分页。
- 菜单与部门页面是明确例外：接口只提供完整列表/树，页面使用后端关键字查询和树形表格，不做伪分页。

### 8.2 查询区

- 常用条件直接显示，低频条件可折叠。
- 提供查询、重置和手动刷新。
- 请求期间避免重复提交，但不阻塞无关区域。
- 空状态、首次加载、刷新加载和请求失败使用不同反馈。

### 8.3 新增与编辑

- 新增和编辑统一使用弹窗。
- 普通表单使用中等宽度弹窗，字段较多时使用大弹窗。
- 角色菜单授权在大弹窗内展示菜单树。
- 删除、重置密码、立即执行、暂停和取消等操作使用确认弹窗。
- 提交成功后关闭弹窗并刷新当前页；提交失败时保留用户输入。

### 8.4 时间与语言

- 仅支持简体中文，Ant Design Vue 使用中文语言包。
- 展示时区固定为 `Asia/Shanghai`。
- 日期时间格式：`YYYY-MM-DD HH:mm:ss`。
- 日期格式：`YYYY-MM-DD`。
- 请求继续传递带时区的 ISO 8601 时间，不在请求层擅自重写。

## 9. API 与类型

### 9.1 类型生成

- 使用 `openapi-typescript` 从 `doc/foundation-frontend-runtime-V3.openapi.json` 生成 TypeScript 类型。
- 提供 `npm run api:types`。
- 生成文件不允许手工修改。
- Axios 请求按领域拆分为薄封装，不生成完整 SDK。
- CI 检查生成类型与 OpenAPI 是否一致。

### 9.2 统一响应与错误

后端响应统一包含 `code`、`message`、`data`、`timestamp` 和 `traceId`。

- 业务失败优先显示后端 `message`。
- 错误反馈中提供可复制的 `traceId`。
- 401 进入统一刷新/退出流程。
- 403 显示无权限提示或页面。
- 404 显示资源不存在。
- 5xx 和网络错误提供重试入口。
- 应用提供全局 Vue 错误兜底页面。
- 首版不接入 Sentry；开发环境保留控制台告警。

## 10. 字典与固定枚举

以 `前端系统字典与枚举接入.md` 为完整契约，关键规则如下：

- 动态字典运行时从字典接口读取，不复制长期常量。
- 多字典页面优先调用批量接口。
- 字典使用同请求去重和最长 5 分钟内存缓存，不写入 `localStorage`。
- 字典管理成功后主动失效对应客户端缓存。
- 字典请求失败时保留原值并提供重试，不伪造默认项。
- 固定枚举在前端维护显示映射；未知值显示原值、中性色标签并记录告警。
- 部门、角色、菜单和任务 Handler 属于关联资源，编辑前从对应接口刷新。

主要固定枚举包括：

- 通用状态：`1=启用`、`0=禁用`。
- 任务配置状态：`1=运行`、`0=暂停`。
- 菜单类型：`DIR=目录`、`MENU=菜单`。
- 菜单显隐：`true=可见`、`false=隐藏`。
- 参数类型：`string`、`boolean`、`number`、`json`。
- 编码重置：`NONE`、`DAILY`、`MONTHLY`、`YEARLY`。
- 异步任务状态使用动态字典 `async_task_status`。

## 11. 页面功能验收

### 11.1 工作台

- 使用 `/api/v1/dashboard/overview` 展示少量核心指标和快捷入口。
- 支持 `/api/v1/dashboard/refresh` 手动刷新。
- 首版不额外引入图表库。

### 11.2 菜单管理

- 树形表格展示目录和菜单。
- 支持关键字查询、新增、编辑、删除。
- 支持父菜单、组件目录、当前路径、图标、排序、显隐和状态配置。
- 菜单类型仅支持 `DIR` 与 `MENU`。

### 11.3 部门管理

- 树形表格展示组织层级。
- 支持关键字查询、新增、编辑和删除。
- 所属上级使用部门树选择。
- 新增根组织时不选择上级部门，请求参数 `parentId` 固定提交为 `0`；选择上级部门时提交对应部门 ID。

### 11.4 角色管理

- 后端分页查询角色。
- 支持新增、编辑、删除和状态维护。
- 通过菜单树配置角色菜单，编辑时读取已选菜单 ID。

### 11.5 用户管理

- 后端分页查询用户。
- 用户列表直接使用查询响应中的部门摘要和多角色摘要，不按行补查关联信息。
- 支持新增、编辑、删除、角色分配和密码重置。
- 部门和角色候选项在打开编辑弹窗时刷新。
- 列表不得展示 `passwordHash`。

### 11.6 系统参数

- 后端分页查询，支持新增、编辑、删除和刷新缓存。
- `configType` 为 `boolean`、`number`、`json` 时在保存前校验内容。
- 系统范围配置显示保护状态，最终修改限制以后端为准。

### 11.7 数据字典

- 字典类型后端分页。
- 字典类型和字典项的多选列独立固定在表格最左侧，批量操作仅提供批量删除。
- 字典项管理支持查询和排序，单条数据可维护启用状态。
- 停用字典前展示影响检查结果。
- 支持 CSV 模板下载、导入、导出、缓存刷新和变更日志。

### 11.8 编码规则

- 后端分页查询，支持新增、编辑和删除。
- 支持按规则编码预览和取下一个编码。
- 重置周期使用固定枚举。

### 11.9 定时任务

- 后端分页查询，支持新增、编辑、删除、立即执行、暂停和恢复。
- Handler 从后端实时读取。
- Cron 使用常用预设加原始表达式输入，不引入大型可视化编辑器。
- 最终 Cron 合法性由后端保存接口校验。
- 支持查看任务执行日志。

### 11.10 异步任务

- 统一登记导入、导出和批量处理任务。
- 后端分页展示任务类型、状态、进度、结果文件和失败原因。
- 支持任务详情、日志、取消和结果下载。
- 页面可见且存在 `PENDING` 或 `RUNNING` 任务时每 5 秒轮询。
- 页面不可见、离开页面或任务全部结束时停止轮询。

### 11.11 文件管理

- 后端分页查询文件资产。
- 支持下载、删除和手工上传。
- 上传使用 `multipart/form-data`；`group`、`remark` 使用查询参数。
- 单文件上限 100 MB，前端和 Nginx 同步限制。
- 不设前端扩展名白名单，最终安全校验由后端完成。
- 只允许上传到 `default`、`documents`、`images`。
- `exports`、`imports`、`attachments`、`generated` 为系统托管分组，按契约限制删除。

### 11.12 日志

- 接口日志、登录日志、操作日志、审计日志均使用后端分页。
- 查询条件以 OpenAPI 为准，包括关键字、时间范围、状态和业务字段。
- 长文本和请求/响应详情使用详情弹窗展示，不挤压列表。
- 登录成功状态使用布尔值；操作类型使用固定枚举。
- 审计日志展示目标对象、操作人、变更摘要及变更前后 JSON。

### 11.13 生命周期治理

- 展示数据保留策略、关联清理任务和治理汇总信息。
- 清理日志使用后端分页，支持策略、目标、状态、关键字和时间范围查询。
- 页面仅提供契约声明的只读治理信息，不提供前端伪造的策略修改或手工清理能力。

## 12. 异步任务轮询

- 列表只有存在进行中任务时才启动 5 秒轮询。
- 详情弹窗打开时同步刷新任务详情和日志。
- 使用 Page Visibility API 在浏览器页面不可见时暂停。
- 保留手动刷新。
- 不在应用全局后台轮询。

## 13. 文件上传契约修正

当前 OpenAPI 将文件二进制错误声明在 `application/json` 下。前端按已确认的真实上传方式实现：

- Content-Type：`multipart/form-data`
- 文件字段：`file`
- 查询参数：`group`、`remark`

后端后续应将 OpenAPI request body 修正为 `multipart/form-data`，以避免生成客户端和文档误用。

## 14. 测试与质量

### 14.1 单元测试

Vitest 至少覆盖：

- token 刷新单飞与失败退出。
- 动态菜单树和路由转换。
- 组件路径自动发现及无效路径降级。
- 字典缓存、请求去重和失效。
- 分页参数和时间格式化。
- 多标签页缓存增删规则。

### 14.2 浏览器冒烟测试

Playwright 使用接口拦截提供确定性数据，覆盖：

- 登录成功和失败。
- 动态菜单加载与无权页面。
- 列表后端分页。
- 一条代表性新增、编辑、删除流程。
- 主题切换和多标签页基础行为。

不追求覆盖率数字，不为纯展示组件编写低价值测试。

## 15. 构建、部署与 CI

### 15.1 Docker

- Dockerfile 使用 Node 构建、Nginx 运行的多阶段镜像。
- 前端与后端运行在不同容器中。
- 两个容器加入同一个外部 Docker 网络。
- 前端 Nginx 将 `/api` 代理到默认 `http://backend:8080`。
- 后端地址允许通过 Compose 环境变量覆盖。
- Nginx 配置 SPA History 回退：`try_files $uri $uri/ /index.html`。
- `client_max_body_size` 设置为 `100m`。

### 15.2 GitHub Actions

每次 push 和 pull request 执行：

1. `npm ci`
2. OpenAPI 生成类型一致性检查
3. TypeScript 类型检查
4. ESLint
5. Vitest
6. 生产构建
7. Docker 镜像构建验证

Playwright 保留独立命令，首版不加入每次 CI。

## 16. 开发与逐页验收流程

严格遵循“一页一确认”：当前页面未获用户确认前，不开发下一菜单页面。

建议顺序：

1. 登录/退出及应用框架
2. 工作台
3. 菜单管理
4. 部门管理
5. 角色管理
6. 用户管理
7. 系统参数
8. 数据字典
9. 编码规则
10. 定时任务
11. 异步任务
12. 文件管理
